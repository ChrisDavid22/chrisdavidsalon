## File Organization Rules

### When Creating New Files:
1. **Reports** ‚Üí `/reports/` (analysis, audits, test results)
2. **Tools** ‚Üí `/tools/` (automation, utilities)
3. **Strategies** ‚Üí `/strategies/` (planning documents)
4. **Documentation** ‚Üí `/docs/` (technical guides, instructions)
5. **Admin Features** ‚Üí `/admin/` (dashboards, admin tools)
6. **Data Files** ‚Üí `/data/` (JSON, persistent storage)
7. **Test Files** ‚Üí `/tests/` (test scripts, test data)
8. **Images** ‚Üí `/images/` (graphics, photos)

### Naming Conventions:
- Documentation: `UPPERCASE-KEBAB-CASE.md`
- HTML files: `lowercase-kebab-case.html`
- JavaScript: `camelCase.js` or `kebab-case.js`
- Data files: `lowercase-kebab-case.json`
